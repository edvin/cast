Wake-on-LAN

The Apple TV app stores this server's MAC address after it connects, and shows a Wake Server button on the "Server Unreachable" screen. Tap it to broadcast a magic packet on the local subnet. For the packet to actually wake a sleeping Windows machine, the server needs all four of the following set up:

  1. BIOS / UEFI — enable the option typically called Wake on LAN, Power on by PCI-E or Resume by PME. The exact wording varies by vendor. You'll need to reboot into setup to change it.
  2. Network adapter driver — open Device Manager → Network adapters → (your NIC) → Properties.
    • On the Power Management tab: enable Allow this device to wake the computer and Only allow a magic packet to wake the computer.
    • On the Advanced tab: find Wake on Magic Packet and set it to Enabled. If there's a Wake on pattern match entry, enable that too.
  3. Disable Windows Fast Startup. This is the single most common reason WoL fails on Windows — Fast Startup puts the NIC into an unpowered state where magic packets go nowhere.
    • Control Panel → Power Options → Choose what the power button does → Change settings that are currently unavailable → uncheck Turn on fast startup → save.
  4. Same local network. The magic packet is a UDP broadcast; it reaches machines on the same subnet as the Apple TV. Cross-subnet WoL requires a router with a "WoL relay" / "forward directed broadcasts" option and is out of scope for this app's button.

Tip: verify WoL works independently first. From another machine on the LAN, try wakeonlan <mac> (Mac/Linux) or any Windows WoL utility while the server is asleep. If that works, the Apple TV button will too.

Folder layout

Every direct subdirectory of your media folder is treated as a TV series (its video files become episodes), except directories named Movies, Films, Film, Filmer or Filmes — those are movie containers. Inside a movie container every video file at any depth is treated as a single movie, regardless of organizational subfolders (e.g. Films/Action/Kill Bill.mkv works the same as a flat file). Place a tmdb.txt with a TMDB ID inside any series folder or movie folder to force a specific match.

Transcoding

MKV / AVI / WebM / FLV files are remuxed (or transcoded, if the codec isn't compatible) to MP4 in the background. The server picks the fastest available H.264 encoder automatically — NVIDIA NVENC, Intel QuickSync or AMD AMF on Windows, VideoToolbox on macOS — falling back to software (libx264) if no hardware encoder works. You can override the choice from the Settings tab, or via the CAST_ENCODER env var.

Logs & diagnostics

The Logs tab shows per-session events only. Turn on Verbose logging in Settings to surface the per-cycle scan chatter, TMDB per-item progress and hardware probe details. The Failures tab lists any remux or TMDB lookup failures with Retry + Delete actions; failures whose underlying file has been removed get swept automatically on the next rescan cycle.
